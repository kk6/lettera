# 利用者の行動シナリオ

## 状態

Draft

## 目的と扱い

この文書は、Letteraを利用する人が、何をきっかけに、どのような順序で行動し、どのような結果を得るかを記述する。後続のUI、機能、データの検討で、利用者の目的や例外時の行動を見失わないための基準とする。

画面、ボタン、メニュー、ダイアログなどの具体的なUIは決定しない。システム内部の処理、Reactの状態、Tauri Command、ファイル形式の実装詳細も扱わない。UIへの対応は[UI設計](ui-design.md)、必要な処理と出力は[機能定義](functions.md)、実装順は[開発ロードマップ](roadmap.md)、個別機能の受け入れ条件は`features/`以下の仕様で扱う。

## 前提

- 当面の利用者は開発者本人のみである。
- 利用者はmacOS上で、プレーンテキストまたはMarkdownの文書を作成・編集する。
- 文書の正本は、利用者が所有するローカルファイルである。
- 一度に編集する文書は一つであり、複数タブと複数ウィンドウは扱わない。
- 利用者の既存ファイルと編集中の内容を失わせないことを最優先する。

## AS-01 思いついた内容を書き始める

### きっかけ

利用者が、思いついた内容を忘れないうちに記録したい。

### 基本の流れ

1. 利用者はLetteraを起動する。
2. 利用者は、ファイルや保存場所を選ぶ前に文章を書き始める。
3. Letteraは、入力された文字、空白、改行、Markdown記法を意図せず変換せず保持する。
4. 利用者は、書いた内容がまだローカルファイルへ保存されていないことを確認できる。

### 完了

利用者が、思いついた内容をLettera上へ記録できている。

### 例外

- 利用者が何も入力しない場合、空の文書のまま利用を続けられる。
- 日本語IMEで変換中の文字列も、確定前に重複または消失しない。

## AS-02 新しい文書を保存する

### きっかけ

利用者が、まだ保存先のない編集中の内容をローカルファイルとして残したい。

### 基本の流れ

1. 利用者は文書の保存を求める。
2. 利用者は保存場所とファイル名を選ぶ。
3. Letteraは現在の内容を選ばれたローカルファイルへ保存する。
4. 利用者は、保存が完了し、現在の内容に未保存の変更がないことを確認できる。

### 完了

編集中の内容がローカルファイルとして保存され、引き続き同じ文書を編集できる。

### 例外

- 利用者が保存先の選択を取りやめた場合、編集中の内容と未保存状態は変わらない。
- 保存に失敗した場合、Letteraは完了したものとして扱わず、編集中の内容を保持する。利用者は再試行、別の場所への保存、または編集の継続を選べる。

## AS-03 既存の文書を開いて編集する

### きっかけ

利用者が、ローカルに保存されているプレーンテキストまたはMarkdown文書を読み、編集したい。

### 基本の流れ

1. 利用者は既存の文書を開くことを求める。
2. 利用者は対象のローカルファイルを選ぶ。
3. Letteraはファイルの内容を読み込む。
4. 利用者は内容を確認し、続きを入力または既存の文章を編集する。
5. 利用者は文書を保存する。
6. Letteraは保存に成功した後、未保存の変更がない状態を示す。

### 完了

選択した文書の編集結果が同じローカルファイルへ保存される。

### 例外

- 利用者がファイル選択を取りやめた場合、操作前の文書をそのまま利用できる。
- 読み込みに失敗した場合、操作前の文書と編集中の内容は変わらない。
- 保存に失敗した場合、編集後の内容と未保存状態を保持する。
- 開く前の文書に未保存の変更がある場合はAS-04へ進む。

## AS-04 未保存の変更がある状態で別の作業へ移る

### きっかけ

利用者が、未保存の変更がある状態で、新規作成、別の文書を開く、文書を閉じる、またはアプリを終了しようとする。

### 基本の流れ

1. Letteraは、現在の文書に未保存の変更があることと、利用者が始めた操作を完了すると内容を失う可能性があることを伝える。
2. 利用者は、保存して続ける、変更を破棄して続ける、または操作を取りやめることを選ぶ。
3. 保存して続ける場合、Letteraは文書を保存する。
4. 保存に成功した後、Letteraは利用者が最初に求めた新規作成、別文書を開く、文書を閉じる、またはアプリ終了を続行する。

### 完了

現在の編集内容について利用者が選択した扱いを適用した後、最初に求めた操作が完了する。

### 代替と例外

- 変更を破棄して続ける場合、Letteraは保存せず、最初に求めた操作を続行する。
- 操作を取りやめた場合、Letteraは現在の文書と編集内容を保持し、元の作業へ戻る。
- 保存先がない場合はAS-02の保存先選択を行い、成功後に元の操作を続行する。
- 保存に失敗または保存先の選択を取りやめた場合、元の操作を続行せず、現在の文書と未保存の内容を保持する。

## AS-05 Markdownの表示結果を確認する

### きっかけ

利用者が、編集中のMarkdown記法が意図した見た目や構造になるか確認したい。

### 基本の流れ

1. 利用者はMarkdownの表示結果を確認できる状態へ切り替える。
2. Letteraは編集中の本文から表示結果を導出する。
3. 利用者は元のMarkdownと表示結果を見比べる。
4. 利用者は必要に応じて本文を修正し、更新された表示結果を確認する。
5. 利用者はMarkdownソースだけを表示する状態へ戻る。

### 完了

利用者が、本文を失わずにMarkdownの表示結果を確認できる。

### 例外

- Markdownとして解釈できない部分があっても、元の本文は保持され、引き続き編集できる。

## AS-06 文書内の文字列を探す

### きっかけ

利用者が、現在の文書に含まれる特定の言葉や記述へ移動したい。

### 基本の流れ

1. 利用者は文書内の検索を始める。
2. 利用者は探したい文字列を入力する。
3. Letteraは一致の有無と、現在参照している一致箇所を示す。
4. 利用者は前後の一致箇所へ移動する。
5. 利用者は目的の箇所を確認または編集する。

### 完了

利用者が、目的の文字列を含む箇所へ移動できる。

### 例外

- 一致する箇所がない場合、Letteraは見つからないことを伝え、本文の状態を変えない。

## AS-07 文書の構造から目的箇所へ移動する

### きっかけ

利用者が、Markdown文書の見出し構造を確認し、目的の節へ移動したい。

### 基本の流れ

1. 利用者は現在の文書のアウトラインを確認する。
2. Letteraは本文から見出しと階層を導出して示す。
3. 利用者は目的の見出しを選ぶ。
4. Letteraは文書内の対応する位置を示す。
5. 利用者はその位置から閲覧または編集を続ける。

### 完了

利用者が、見出しを手がかりに目的の箇所へ移動できる。

### 例外

- 文書に見出しがない場合、Letteraは移動先がないことを示す。

## AS-08 最近使った文書を再び開く

### きっかけ

利用者が、以前Letteraで開いた文書の作業を再開したい。

### 基本の流れ

1. 利用者は最近使った文書を確認する。
2. 利用者は、文書を識別できる名前または場所を手がかりに対象を選ぶ。
3. Letteraは参照先のローカルファイルを読み込む。
4. 利用者は文書の内容を確認し、閲覧または編集を続ける。

### 完了

利用者が、ファイルの場所を選び直さずに過去の文書を開ける。

### 例外

- 開く前の文書に未保存の変更がある場合はAS-04へ進む。
- 参照先が移動、削除、または読み込み不能の場合、Letteraは開けなかったことを伝える。現在の文書は変更しない。

## AS-09 フォルダーから文書を探して開く

### きっかけ

利用者が、関連する文書を含むローカルフォルダーから対象を探したい。

### 基本の流れ

1. 利用者は参照するフォルダーを選ぶ。
2. Letteraはフォルダー以下の構成を示す。
3. 利用者は階層をたどり、対象のテキストファイルを選ぶ。
4. Letteraはファイルを読み込む。
5. 利用者は内容を閲覧または編集する。

### 完了

利用者が、フォルダーの構成を手がかりに目的の文書を開ける。

### 例外

- 開く前の文書に未保存の変更がある場合はAS-04へ進む。
- フォルダーまたはファイルを読み込めない場合、Letteraは失敗を伝え、現在の文書を変更しない。

## AS-10 ファイル操作の失敗から回復する

### きっかけ

利用者が求めたファイルまたはフォルダーの読み込み、あるいは文書の保存が完了しなかった。

### 基本の流れ

1. Letteraは完了しなかった操作を伝える。
2. Letteraは、現在の文書と編集中の内容が保持されているかを伝える。
3. 利用者は、失敗した操作の再試行、別の対象や保存先の選択、または元の編集作業へ戻ることを選ぶ。
4. Letteraは選ばれた操作を行い、成功するまで元の文書を不用意に置き換えない。

### 完了

利用者が編集内容を失わず、成功可能な別の行動または元の作業へ移れる。

## AS-11 好みの操作方法でコマンドを実行する

### きっかけ

利用者が、文書の保存などの主な操作を現在の作業に合う方法で実行したい。

### 基本の流れ

1. 利用者は、画面上の操作、キーボードショートカット、またはmacOSメニューから目的のコマンドを選ぶ。
2. Letteraは、どの入口を使った場合も同じ文書状態に対して同じ操作を実行する。
3. 利用者は操作の結果を同じ方法で確認できる。

### 完了

利用者が入口の違いを意識せず、目的の操作を完了できる。

### 例外

- 現在の状態で実行できない操作は、実行可能であるかのように示さない。

## AS-12 表示設定を変更する

### きっかけ

利用者が、本文の文字を読みやすい大きさにする、または周囲の明るさやmacOSの外観に合うカラーモードを使いたい。

### 基本の流れ

1. 利用者はLetteraの設定を確認する。
2. 利用者は基本フォントサイズを変更する。
3. Letteraは選ばれたフォントサイズを編集画面へ反映する。
4. 利用者はカラーモードを、システム設定に従う、ライト、ダークから選ぶ。
5. Letteraは選ばれたカラーモードをアプリUIへ反映する。
6. 利用者はLetteraを終了し、後から再び起動する。
7. Letteraは保存済みの設定を読み込み、画面へ反映する。

### 完了

利用者が選んだ基本フォントサイズとカラーモードが画面へ反映され、アプリの再起動後も維持される。

### 例外

- 保存済みの設定がない場合、Letteraは既定値を使用する。
- 保存済みの値が読み込めない、または利用できない場合、Letteraは安全な既定値で起動し、文書の編集を妨げない。
- カラーモードがシステム設定に従う場合、macOSの外観が変わるとLetteraの外観も追従する。

## 現段階で決めないこと

- 起動時に前回の文書を復元するか
- 最近使った文書を何件、どの期間保持するか
- フォルダーをまたいだ検索や全文検索を提供するか
- MarkdownプレビューとSource editorのスクロールを同期するか
- ショートカットの具体的なキーとメニュー階層
- 基本フォントサイズとして選べる最小値、最大値、変更単位
- 設定画面に将来追加する項目と分類
- 外部アプリによるファイル変更を検出し、どのように調停するか

これらは、対応する利用目的へ着手した時点で、必要なシナリオを追加または更新してからUIと機能へ反映する。
