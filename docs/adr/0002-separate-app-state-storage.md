# ADR-0002: アプリ固有状態をユーザー文書と分離して保存する

## Status

proposed

## Date

2026-07-22

## Context (Why)

Letteraでは、利用者が選んだ基本フォントサイズとカラーモードを、アプリの終了と再起動をまたいで維持する必要がある。また、最近使った文書を、ファイルの場所から探し直さずに再び開ける必要がある。

これらはLetteraの利用体験を支える状態だが、利用者が作成するプレーンテキストまたはMarkdown文書の内容ではない。設定や履歴をユーザー文書へ書き込むと、一般的なテキストファイルとの互換性が損なわれ、Lettera固有の情報が利用者の正本へ混入する。

一方、現時点で保持する状態は少数の設定値と文書への参照であり、複雑な検索やエンティティ間の関係を必要としない。

## Intent (What)

- 基本フォントサイズとカラーモードを、ユーザー文書とは独立したLettera固有のアプリデータとして保持する。
- カラーモードは、システム設定に従う、ライト、ダークの3種類を扱う。
- 保存済みの設定を起動時に読み込み、現在の画面へ反映する。
- 最近開いた文書と、新規保存に成功した文書への参照を、最近使った文書として保持する。
- 設定または履歴の保存失敗によって、編集中の本文やユーザー文書を失わせない。
- 保存済みの設定が存在しない、壊れている、または利用できない場合も、安全な既定値で文書を編集できる状態にする。
- 将来設定項目が増えた場合に、既存の設定を読み継げるようにする。

## Constraints

- ユーザー文書へLettera固有の設定や履歴を混入させない。
- アカウントやクラウドサービスを必要とせず、ローカル環境だけで利用できる。
- アプリ固有データの破損や欠落によって、Letteraの起動や文書編集を不可能にしない。
- ファイルパスなどの外部入力は、ネイティブ境界で信頼せずに扱う。
- 現在必要な少数の設定と参照のために、不必要に複雑な保存基盤を導入しない。

## Alternatives Considered

### ユーザー文書へ設定や履歴を埋め込む

文書と一緒に情報を保持できるが、プレーンテキスト／Markdownとの互換性を損ない、利用者の正本へLettera固有の情報が混入するため採用しない。

### アプリ終了時に状態を破棄する

保存機能が不要になるが、起動のたびに表示設定をやり直す必要があり、最近使った文書から作業を再開できないため採用しない。

### リレーショナルデータベースを必須とする

将来の高度な検索や関係管理には適する可能性があるが、現在の少数の設定値と文書参照に対して、スキーマ管理や依存関係のコストが大きいため現時点では採用しない。

## Consequences

### 良い影響

- ユーザー文書の可搬性とLettera以外のツールとの互換性を維持できる。
- 利用者が選んだ表示設定を再起動後も利用できる。
- 最近使った文書から作業を再開できる。
- 設定や履歴が破損しても、既定値によって文書編集を継続できる。
- 現在の規模に合った軽量な保存方法を選択できる。

### トレードオフ

- ユーザー文書とは別に、アプリ固有データの読込、検証、保存、スキーマ更新を管理する必要がある。
- アプリ固有データを削除すると、設定と最近使った文書の履歴は失われる。
- 将来、全文検索や多数の関連データが必要になった場合は、保存方式を再検討する可能性がある。
