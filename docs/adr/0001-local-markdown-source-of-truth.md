# ADR-0001: ローカルMarkdownを文書の正本とする

## Status

proposed

## Date

2026-07-21

## Context (Why)

Letteraで作成した文書は、特定のアプリケーションやサービスが利用できなくなっても、利用者が継続して所有し、ほかの一般的なツールから読み書きできる必要がある。

エディタ内部の都合を優先した独自形式を正本にすると、アプリへの依存、変換時の情報損失、文書を直接扱えない状況が生じる。また、複数の編集モードがそれぞれ別の文書表現を持つと、モード切り替えによる内容の不一致や損失が起こり得る。

## Intent (What)

- 保存済み文書の正本を、ローカルのプレーンテキストまたはMarkdownファイルとする。
- 文書はLettera以外の一般的なツールからも読み書きできる状態を保つ。
- Source、Split、将来のSeamlessは、一つの論理文書に対する異なる表示・編集方法とする。
- プレビュー、アウトライン、編集用の内部表現は、正本から導出できる状態にする。
- 内部表現を、利用者の明示的な選択なしに保存ファイルへ混入させない。

## Constraints

- アカウント、クラウドサービス、独自データベースを主要機能の必須条件にしない。
- モード切り替えによってMarkdownの内容を失わせない。
- 変換または正規化によって内容が変わる場合は、利用者から観察できる仕様として扱う。
- 保存や変換の失敗時に、既存ファイルと編集中の内容を可能な限り保持する。

## Alternatives Considered

### 独自の文書形式を正本とする

高度な編集状態をそのまま保存しやすいが、Letteraへの依存が生じ、ほかのツールとの互換性が低下するため採用しない。

### エディタライブラリの内部モデルを正本とする

ライブラリ上の実装は単純になる可能性があるが、将来のライブラリ変更が文書形式へ影響し、Markdownとの往復変換で情報を失う可能性があるため採用しない。

### 編集モードごとに別の文書状態を持つ

各モードを独立して実装しやすいが、内容の同期、Undo履歴、保存対象が複雑になり、モード切り替え時の不一致を招くため採用しない。

## Consequences

### 良い影響

- 利用者が文書を長期的に所有できる。
- Lettera以外のツールとの相互運用性を保てる。
- 編集モードを切り替えても、保存対象を一つに保てる。
- 特定のエディタライブラリへの依存を文書形式から切り離せる。

### トレードオフ

- Seamless編集などの高度な内部表現をMarkdownへ安全に戻す設計が必要になる。
- Markdownで表現できない編集状態は、正本とは別の一時状態として扱う必要がある。
- 変換ライブラリを選ぶ際に、Markdownの往復保持を評価する必要がある。
